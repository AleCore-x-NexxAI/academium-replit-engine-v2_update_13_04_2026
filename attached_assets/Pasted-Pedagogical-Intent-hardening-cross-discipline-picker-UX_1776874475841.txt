Pedagogical Intent hardening, cross-discipline picker UX, framework-anchoring coreConcepts, superlative expansion, auto-inference short-circuit
Context
The wizard (from commits 060df97 and 41d1f83) renders the right fields — discipline accordion picker with cross-discipline counter, "Other" per discipline, professorNotes, courseContext, reasoningConstraint, DecisionDimensionsEditor — but it does not enforce the required-field rules we decided in the call. The frameworkInference.ts subsystem exists but is auto-invoked; we are explicitly dropping auto-inference (professors choose). The buildIntentBlock in canonicalCaseGenerator.ts builds the framework anchoring block but does not surface each framework's coreConcepts, which §9.2 of the packet requires. The narrator's PROHIBITED_PATTERNS list is missing the §3.4 expansion. The cross-discipline picker has a UX bug: toggling a framework that's already selected in another discipline removes it entirely instead of showing it as disabled with a tooltip.
This task fixes all of the above as a single coherent change set.
Scope
A1. Wizard required-field enforcement (client + server)
Target file: client/src/components/CanonicalCaseCreator.tsx.
Define REQUIRED_FIELDS rules:

teachingGoal: required, minimum 20 trimmed characters.
selectedDisciplines.length >= 1: required. A discipline alone (no specific framework selected within it) is valid. You cannot proceed with zero disciplines.
selectedFrameworks: optional. A professor may select one or more specific frameworks from within any selected discipline, or select none and let the generator pick from the discipline pool. Do not require any specific framework.
courseContext: required, minimum 20 trimmed characters.
reasoningConstraint: optional. No minimum length.
professorNotes: optional, existing 500-char cap preserved.
decisionDimensions (from DecisionDimensionsEditor): optional.

UI behavior:

Disable the "Generate" button whenever any required field is unmet. Do not submit.

For teachingGoal: EN "Teaching goal is required (at least 20 characters)." / ES "El objetivo pedagógico es obligatorio (mínimo 20 caracteres)."
For disciplines: EN "Select at least one discipline." / ES "Selecciona al menos una disciplina."
For courseContext: EN "Course context is required (at least 20 characters)." / ES "El contexto del curso es obligatorio (mínimo 20 caracteres)."


The existing framework counter cap (MAX_FRAMEWORKS) stays.
Route the error strings through client/src/lib/translations.ts using new keys canonicalCase.validation.teachingGoalRequired, canonicalCase.validation.disciplineRequired, canonicalCase.validation.courseContextRequired. Add both EN and ES.
