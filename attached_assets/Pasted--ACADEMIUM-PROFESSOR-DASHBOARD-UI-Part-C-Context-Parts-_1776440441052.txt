## C2.1 — Top stats row (4 cards)

**Layout:** 4 equal-width white cards in a row. `grid-template-columns:repeat(4,1fr); gap:10px; margin-bottom:16px`. Each card: `background:#fff; border:0.5px solid rgba(0,0,0,0.1); border-radius:12px; padding:14px`.

**Card structure:**
- Large number on top (`font-size:22px; font-weight:500; color:#1a1a1a`)
- Label (`font-size:11px; font-weight:500; color:#1a1a1a`)
- One-line italic description (`font-size:11px; color:#999; font-style:italic; line-height:1.4`)

**Four cards:**

| Card | Value source | Description below value |
|---|---|---|
| Completed | Count of sessions where `session_status === "complete"` | "Number of students who have finished all turns and submitted the reflection" |
| In progress | Count of sessions where `session_status === "active"` | "Number of students who have started but not yet completed the simulation" |
| Biggest drop point | Computed (see below) | "The turn where average reasoning depth dropped most significantly across the class" |
| Applied course theory | Computed (see below) | "How many students showed evidence of applying at least one professor-specified framework" |

**Biggest drop point computation:**
```
For each turn N (2 to max):
  avgN = mean of student.turns[N].rds_band across all completed students
  (where Surface=1, Engaged=2, Integrated=3)
  delta[N] = avgN - avg[N-1]

Largest negative delta → "Turn N"
If no negative delta exists → "—"
If no completed students yet → "—"
```

**Applied course theory count:**
- Count students with ≥1 `framework_detections[].level === "explicit"` OR `"implicit"` across any turn of their completed session.
- Display format: "N of M" where M = completed students, N = students with applied theory.
- If no frameworks configured on scenario → display "—".
- If 0 completed students → display "—".

**API endpoint:** `POST /api/scenarios/:scenarioId/class-stats`
```json
Returns: {
  "completed": 3,
  "inProgress": 1,
  "biggestDropPoint": { "turn": 3, "delta": -0.8 } | null,
  "appliedCourseTheory": { "n": 2, "m": 3 } | null
}
```

Authorization: professor only, only the owning professor.

  - ❌ "You should address this gap in your next lecture."

- Body bg: `#f5f5f3`

- Font sizes: 10px–22px range (most common: 11px, 12px, 13px)

**Status color palette (use ONLY these colors for status indicators):**

| State | Hex | Bg | Text |
|---|---|---|---|
| Integrated / Transferring / Demonstrated / Positive | `#1D9E75` | `#EAF3DE` | `#27500A` |

| Absent / Negative | `#D85A30` | `#FCEBEB` | `#791F1F` |

## C4.2 — Empty and edge states

- Class overview with 0 completed sessions: Show empty state message "No sessions completed yet. Dashboard data will appear once students finish simulations." Don't render charts.
- Class overview with 1 completed session: all sections render correctly with N=1 data.
- Scenario with 0 frameworks configured: Module Health section shows "No frameworks configured for this case. Add frameworks in the scenario editor to see how students apply them." Hide framework grid, debrief opener, and deeper view.
- Biggest drop point when depth never dropped: show "—".
- Radar chart with all-zero signal values: renders as a single center point, no crash.
- Student in progress: "View session" disabled with `opacity:0.35; cursor:default`. "Arc" column shows "Available when completed".
- Student with no KPI movements at all (all BLOCKs): KPI trajectory section shows "No KPI movements recorded for this session."

## C4.3 — Language compliance

Every AI-generated dashboard text respects scenario language:
- English scenarios → all AI output in English
- Spanish scenarios → all AI output in Spanish
- UI chrome (static labels, buttons, tab names) uses the professor's UI language from the existing `useTranslation()` hook

## C4.4 — Performance targets

- Aggregation endpoints (`class-stats`, `module-health`, `depth-trajectory`, `class-patterns`, `students-summary`) return in < 500ms for a class of 30 students.
- `dashboard_summary` is NOT recomputed — already stored from B4.
- Module health, class patterns, depth chart annotations: cached per-scenario for 5 minutes. Invalidate when: (a) a new session completes, (b) the scenario is edited.
- Student modal tabs: fetch on-demand from stored session state, no cache needed.

## C4.5 — Authorization

- All dashboard endpoints behind professor-role auth.
- Only the owning professor accesses their scenario's dashboard.
- Students NEVER see the dashboard or its endpoints (return 403).
