1. Inappropriate Input Handling – Wrong Behavior

What actually happened (exact)
	•	As a professor, I typed “idiota”.
	•	The system responded:
“Parece que hubo un problema con tu entrada…”
	•	Then I typed “pendejo”.
	•	The system responded:
“Parece que tu última entrada no está relacionada con el contexto del caso de estudio…”
	•	After that, the system moved on to the next decision as if nothing happened.

Why this is wrong
	•	The system ignored the inappropriate language.
	•	It did not enforce any guardrails.
	•	It allowed the simulation to continue without a valid response.
	•	It treated the input as noise instead of correcting behavior.

Required fix

When a user enters offensive or inappropriate language or anything not relevant:
	•	Do NOT advance to the next decision
	•	Show a clear warning message, for example:
	•	Remind the user that inappropriate language is not allowed or say your response is not relevant
	•	Ask them to provide a proper response related to the case. Remember the checking agent please
	•	Block progression until:
	•	The user submits a meaningful, relevant answer
	•	The simulation must stay on the same decision step

This applies to:
	•	Professor testing mode
	•	Student mode
	•	Any free-text input
