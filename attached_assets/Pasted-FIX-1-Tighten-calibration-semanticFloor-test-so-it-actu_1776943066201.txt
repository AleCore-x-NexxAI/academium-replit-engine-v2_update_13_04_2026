FIX 1 — Tighten calibration.semanticFloor test so it actually catches regressions
Context
The server/__tests__/calibration.semanticFloor.test.ts file contains a local applySemanticFloor helper function that re-encodes the Tier-2 semantic floor logic. The problem: when inputWordCount < 10, the test helper returns not_evidenced directly, but the real code in server/agents/frameworkDetector.ts:345-376 falls through to Tier 3 (signal-pattern) first, and only pushes not_evidenced if Tier 3 also fails. If someone accidentally breaks the Tier-3-fallthrough in the real code, this test would still pass — a false-negative regression risk.
The other two calibration tests (calibration.moduleHealth.test.ts and calibration.appliedCourseTheory.test.ts) also re-encode production logic in local helpers rather than importing from routes.ts. They are byte-identical mirrors today, but the same regression risk applies.
The real code for appliedCourseTheory and moduleHealth is embedded in large handler bodies in server/routes.ts. We can extract the pure scoring/filtering logic into a standalone module (like what was already done for mergeReflectionAnalytics.ts), then import it both in routes.ts and in the tests.
