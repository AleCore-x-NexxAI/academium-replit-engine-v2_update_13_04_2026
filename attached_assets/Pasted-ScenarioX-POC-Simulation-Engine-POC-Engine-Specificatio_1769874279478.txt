Purpose of this Document
Why this matters:
This document locks the POC version of ScenarioX’s simulation engine. The POC is not trying
to prove every feature. It’s proving the core loop:
decision → world responds → metrics move with clear causality → next decision
becomes more informed.
This spec is intentionally rigid. Anything not explicitly included is out-of-scope for POC.

● Students must feel the simulation is simple, calm, and fair.
● The system must never imply the student is “stupid,” “wrong,” or “failed.”
● Every response must feel like: “You’re driving. I’m the instrument.”

3) Research-Anchored Foundations (POC Translation)
Why this matters:
This is the “defensibility bridge” — it forces the POC behavior to match the canon, without
adding complexity.
3.1 Pedagogical foundations embodied in POC behavior (named)
● Active Learning: decisions drive the experience (not passive content).
● Experiential Learning (Kolb): each turn includes consequence + reflection cue.
● Constructivism: the engine probes reasoning, not recall.
● Microlearning / Chunking: 3-turn structure prevents overload and protects completion.
● Scaffolding / ZPD: Turn 1 reduces cognitive load; Turns 2–3 increase depth.
3.2 Psychological foundations protected in POC behavior (named)
● Cognitive Load Theory: Turn 1 is MC onboarding; UI is consistent; prompts are short.
● Self-Efficacy: low-effort handling is supportive and structured (no shame).
● Psychological Safety: profanity/trolling is redirected; tone stays academic.
● Priming/Framing: no grading cues; no scoreboard vibe; no “right answer” language.

● Tier 0: 0 change

● Tier 2: ±4 to ±7
● Tier 3: ±8 to ±12 (rare; must be event-level)

9) Hint System
Why this matters: Hints can quietly turn ScenarioX into a tutor. In the POC, the default must
preserve student ownership and professor authority.
9.1 Default State (Hard)
● Hints are OFF by default in the POC deployment.
● Hints can be enabled explicitly (toggle) for the POC if the professor wants additional
scaffolding.
9.2 What Hints Are Allowed To Do (When Enabled)
● Clarify constraints, stakeholders, objective, and what information is missing.
● Provide framing questions or frameworks (e.g., “What are the tradeoffs?”).
● Encourage structured justification.
9.3 What Hints Must Never Do
● Provide “the best action” or “recommended choice.”
● Suggest what the professor wants.
● Reveal “correct answers.”
● Become a “Suggested Actions” panel.
9.4 Behavior When Hints Are OFF (Default)
If a student is stuck or asks for help:
● The engine uses Acknowledge → Gap → Prompt and/or redirects:
○ “Ask your professor for guidance on course content; I can help you structure your
reasoning.”
10) Branching Model (POC)
POC uses soft branching:
● same main storyline
● consequences shift via:
○ metric values
○ stakeholder tone
○ urgency
○ constraints in next turn
No dead ends. Bad decisions create harder constraints, not termination.

POC is successful if:
1. Turn structure is consistent every time.
2. Metric movement is explainable with “why + why this magnitude.”
3. No answer-giving UI (“Suggested actions” removed).

6. System feels fair, calm, and in control.
7. The engine successfully redirects profanity/trolling without escalation.
8. Spanish output is consistent (no English leakage).
